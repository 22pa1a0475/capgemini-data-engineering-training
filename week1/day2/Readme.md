Week 1 Day 2 – Joins and GROUP BY Practice

1. Objective
   In this session, the objective is to understand and practice JOIN operations and GROUP BY aggregations separately using SQL and PySpark.

2. Problem Summary
   This day consists of two separate practice modules:

Joins Practice:
Understanding different types of joins such as inner join, left join, right join, and full join
Analyzing matching and non matching records between tables

GROUP BY Practice:
Performing aggregation operations such as sum, count, average, minimum, and maximum
Applying grouping logic on datasets
Using conditions with WHERE and HAVING

3. Approach
   Worked on joins and group by as separate exercises

Joins Practice:
Solved queries using different join types
Validated results based on matching conditions
Analyzed null and unmatched records

GROUP BY Practice:
Solved aggregation queries on given datasets
Grouped data based on required columns
Applied HAVING clause for aggregated filtering

4. Key Transformations Used
   JOIN operations for combining datasets
   GROUP BY for aggregation
   SUM, COUNT, AVG, MIN, MAX for calculations
   WHERE for filtering records
   HAVING for filtering grouped results

5. Data Engineering Considerations
   Used correct join keys to ensure accurate results
   Handled null values in join outputs
   Maintained proper grouping columns
   Clearly differentiated WHERE and HAVING usage

6. Challenges Faced
   Understanding behavior of different join types
   Writing correct GROUP BY queries
   Separating join and aggregation logic
   Debugging query errors

7. Learnings
   Strong understanding of join operations
   Clear knowledge of GROUP BY and aggregation
   Improved ability to work with relational datasets
   Better problem solving skills in SQL and PySpark
