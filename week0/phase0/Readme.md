Capgemini Data Engineering Training Portfolio

Overview
This repository presents my complete learning journey during the Capgemini Data Engineering Training. It covers SQL, PySpark, Databricks, Delta Lake, and end to end data pipeline development. The training is structured across four weeks, progressing from foundational concepts to real world data engineering projects and production level pipelines.

Week 0 Foundations

This week focused on completing foundational Databricks learning modules to understand core data engineering concepts. The modules covered data ingestion, workflow automation, pipeline development, and DevOps essentials for data engineering.

Through these modules, I learned how data is ingested into systems, how workflows are scheduled and automated, and how pipelines are built using Spark-based technologies. This week provided a strong conceptual understanding of modern data engineering platforms and prepared me for hands-on practice in the following weeks.

Week 1 Advanced SQL and PySpark

This week focused on strengthening SQL and PySpark skills through structured practice and problem solving.

I learned different types of joins and how to apply group by with aggregation functions. I implemented conditional logic using case when and performed advanced analysis using window functions such as row number, rank, dense rank, lag, and lead. I also worked with date functions, string manipulation, null handling, and regex processing.

I completed practical tasks such as student data validation, duplicate detection, and classification using joins and window functions. I also executed a complete data pipeline within a time constraint, which improved debugging and validation skills.

In addition, I worked on real world projects including an e commerce data pipeline and a car sales analytics pipeline. These projects involved multi table joins, data cleaning, transformation, and generating business insights.

Week 2 Core Data Engineering Concepts

This week focused on building strong data engineering fundamentals and designing structured pipelines.

I practiced SQL concepts such as subqueries and common table expressions to write efficient and readable queries. I also worked with window functions like lead and lag for trend analysis and sequential data comparison.

I developed an insurance data pipeline where I performed data cleaning, validation, transformation, and analysis to generate insights such as customer risk scores and agent performance.

I implemented Delta Lake features including insert, update, delete, and merge operations, along with time travel and version history. This helped in understanding data reliability, ACID transactions, and debugging using historical data.

I also implemented Medallion Architecture by building bronze, silver, and gold layers. Raw data was ingested in the bronze layer, cleaned and transformed in the silver layer, and aggregated into business insights in the gold layer.

Week 3 Real World Data Engineering Projects

This week focused on applying all learned concepts to real world projects through individual and group work.

As an individual project, I built an end to end data pipeline using PySpark and DBT, following Medallion Architecture and generating key performance indicators.

In group projects, I worked on a retail lakehouse pipeline using Delta Live Tables. This included data ingestion, cleaning, implementing slowly changing dimensions, and generating business insights with data quality checks.

I also contributed to a banking analytics project that involved integrating multiple datasets, performing transformations, applying validation rules, and generating insights for business decision making.

Additionally, I worked on a healthcare data pipeline in a smaller team, focusing on handling domain specific datasets and generating meaningful insights.

This week provided experience in collaborative development, real world datasets, and production level pipeline design.
