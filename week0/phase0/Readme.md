Capgemini Data Engineering Training Portfolio

Overview
This repository presents my complete learning journey during the Capgemini Data Engineering Training. It covers core and advanced concepts in SQL, PySpark, Databricks, Delta Lake, and end to end data pipeline development. The training is structured across four weeks, starting from fundamentals and progressing to real world data engineering projects and production level pipelines.

Week 0 Foundations
This week focused on building a strong foundation in PySpark and Databricks. I worked with DataFrame operations such as filtering, selecting columns, grouping, and aggregation. I also understood the differences between SQL and PySpark approaches. Later, I implemented basic ETL workflows, performed data cleaning, handled missing values, and generated simple analytical outputs. This week helped me understand how data flows through a pipeline and how transformations are applied step by step.

Week 1 Advanced SQL and PySpark
In this week, I strengthened my SQL and PySpark skills through structured practice. I learned different types of joins and how to apply group by with aggregations. I implemented conditional logic using case when and performed advanced analysis using window functions such as row number, rank, dense rank, lag, and lead. I also worked with date functions, string manipulation, null handling, and regex processing.
I completed practical tasks such as student data validation and duplicate detection using window functions. I also executed a complete pipeline under time constraints to improve debugging and problem solving skills. Additionally, I worked on real world projects including an e commerce pipeline and a car sales analytics pipeline, which involved multi table joins, data cleaning, and business insights generation.

Week 2 Core Data Engineering Concepts
This week focused on strengthening core data engineering concepts and building structured pipelines. I practiced SQL using subqueries and common table expressions to improve query readability and efficiency. I learned window functions like lead and lag for trend analysis and sequential data comparison.
I built an insurance data pipeline where I performed data cleaning, validation, transformation, and analysis to generate meaningful insights such as customer risk scores and agent performance. I also implemented Delta Lake features including insert, update, delete, merge operations, and time travel to understand data versioning and reliability.
Finally, I implemented Medallion Architecture with bronze, silver, and gold layers, where I ingested raw data, cleaned and transformed it, and generated business level insights. This week helped me understand how scalable and reliable data pipelines are designed.

Week 3 Real World Data Engineering Projects
This week focused on applying all learned concepts in real world scenarios through individual and group projects.
As an individual project, I built an end to end pipeline using PySpark and DBT, following Medallion Architecture and generating key performance indicators.
In group projects, I worked on a retail lakehouse pipeline using Delta Live Tables, where I implemented data ingestion, cleaning, SCD Type 2, and KPI generation with data quality checks. I also contributed to a banking analytics project that involved integrating multiple datasets, performing transformations, applying validation rules, and generating insights for decision making.
Additionally, I worked on a healthcare data pipeline in a smaller team, focusing on handling domain specific data and generating useful insights. This week provided experience in collaborative development and production level pipeline design.

Technologies Used
PySpark, SQL, Databricks, Delta Lake, DBT, Delta Live Tables, AWS S3

Skills Developed
Data cleaning and validation, ETL pipeline development, data transformation and aggregation, advanced SQL and window functions, Medallion Architecture implementation, Delta Lake features such as ACID transactions and time travel, real world data analysis, and teamwork in data engineering projects

Conclusion
This training provided hands on experience in modern data engineering practices. I developed the ability to design and build scalable data pipelines, process large datasets, and generate meaningful business insights. It also improved my problem solving skills and prepared me for real world data engineering roles.
